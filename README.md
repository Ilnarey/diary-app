- удаление# 📖 Личный дневник

Простое веб-приложение для ведения личного дневника, разработанное в рамках учебной практики по теме **«Построение CI/CD-пайплайна»**.

---

## Описание

Приложение позволяет создавать, просматривать, редактировать и удалять записи личного дневника.

Реализованные возможности:

- ✏️ создание записи;
- 📋 просмотр всех записей;
- 📝 редактирование записи;
- 🗑 удаление записи.

### REST API

| Метод | Endpoint | Назначение |
|--------|----------|------------|
| GET | `/entries` | Получить список записей |
| POST | `/entries` | Добавить новую запись |
| PUT | `/entries/:id` | Изменить запись |
| DELETE | `/entries/:id` | Удалить запись |

## Запуск локально

### 1. Клонировать репозиторий

```bash
git clone https://github.com/ВАШ_ЛОГИН/diary-app.git
```

### 2. Перейти в папку проекта

```bash
cd diary-app
```

### 3. Установить зависимости

```bash
npm install
```

### 4. Запустить приложение

```bash
npm start
```

После запуска приложение будет доступно по адресу

```
http://localhost:3000
```

## Запуск в Docker

### Сборка Docker-образа

```bash
docker build -t diary-app .
```

### Запуск контейнера

```bash
docker run -p 3000:3000 diary-app
```

После запуска приложение будет доступно по адресу

```
http://localhost:3000
```

Если используется Docker Compose:

```bash
docker compose up --build
```

## CI/CD

Для автоматизации процессов разработки используется **GitHub Actions**.

### Continuous Integration (CI)

При каждом Push и Pull Request автоматически выполняются:

- установка зависимостей;
- проверка проекта;
- запуск тестов;
- сборка Docker-образа.

При возникновении ошибок процесс завершается неуспешно, что предотвращает попадание некорректного кода в основную ветку проекта.

### Continuous Deployment (CD)

После успешного обновления ветки **main** приложение автоматически разворачивается на сервере Render (или Railway) и становится доступным по публичному URL.
